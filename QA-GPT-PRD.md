# QA-GPT PRD: AI 기반 IT SI 산출물 자동 품질검토 플랫폼

## Problem Statement

IT SI 프로젝트에서 산출물 품질관리가 구조적 한계를 가지고 있습니다:

- **검토 프로세스 비효율성**: 제3자 QA, PMO, PM 자체 검토에만 의존하며, 검토 시점이 지연되고 전수 검토가 불가능합니다.
- **비용 및 시간 소모**: 검토 소요 시간이 수일에 달하고, 검토 인력과 비용이 과다합니다.
- **PM 역량 의존도 심화**: 산출물 품질이 담당 PM의 경험과 역량에 과도하게 의존하여 품질 편차가 크고, 신규/중간 투입 PM의 결함 발생 빈도가 높습니다.
- **후행 비용 증가**: 결함 발견이 늦어지면서 수정 비용(Cost of Change)이 기하급수적으로 증가하고, 검수·감리 단계에서 대규모 수정을 요구받습니다.
- **실시간 검증 체계 부재**: 결함의 70% 이상이 요구사항 단계 누락에서 기인하지만, 제출 시점에 자동 검증하는 체계가 없습니다.

## 핵심 네러티브

우리 팀은 3명으로 이루어져 있습니다 — 10년차 숙련자 1명, 3년차 이하 신입 PM 2명. 신입 PM들이 요구사항정의서나 설계서를 작성할 때 피드백을 받으면 실제로 배우고 개선되는 모습을 봅니다. 그런데 현재는 모든 검토가 외부 제3자 QA에 의존하기 때문에 검수가 수일 소요되고, 결국 리스크를 뒤늦게 발견하게 되며, 최악의 경우 검수 단계 실패로 오픈이 지연되는 사태까지 일어납니다. 

**QA-GPT는 이 악순환을 다음과 같이 끊습니다**: 신입 PM이 산출물을 업로드하면 수 분 내 3가지(검토 결과, 개선본, 비교표)를 자동으로 생성해주고, 신입 PM이 AI 개선본을 검토하면서 "왜 이렇게 개선해야 하는가"를 즉시 배울 수 있습니다. 무제한 재검증으로 신입도 스스로 품질을 높일 수 있고, 고객사에는 AI가 검토한 근거를 담은 비교표를 제출해서 신뢰도를 확보합니다. 결과적으로 신입 PM의 역량이 빨리 올라가고, 검수 실패로 인한 오픈 지연 같은 리스크가 사라지며, 팀 전체의 산출물 품질이 일관되게 유지됩니다.

## Solution

**QA-GPT**는 AI 기반의 자동 품질검토 및 개선 플랫폼입니다. PM이 산출물을 업로드하면, AI가 다음 3가지를 자동으로 생성합니다:

1. **품질검토 결과서**: ERROR/WARNING/OK 3단계로 분류된 검토 결과 및 점수(0~100)
2. **AI 개선 원본 파일**: 결함을 직접 수정한 산출물 새 버전 ([개선] 태그 표시)
3. **개선 비교표**: Before/After 항목별 비교 및 개선 사유 문서화

이를 통해 PM은 즉시(수 분 내) 품질 피드백을 받고, AI의 개선 원본을 참고하여 최종 산출물을 결정할 수 있습니다. 무제한 재검증이 가능하고, 고객사 검수 시 투명한 근거 자료를 제출할 수 있습니다.

## User Stories

### PM (프로젝트 매니저) 관점

1. 산출물 작성 후 QA-GPT에 업로드하여, 수 분 내에 품질검토 결과를 받고 싶다.
2. AI가 제시한 개선 원본을 참고하여, 어떻게 개선해야 하는지 학습하고 최종 산출물을 결정하고 싶다.
3. 수정 후 재업로드하여 무제한으로 품질을 재검증하고, 점수 개선을 확인하고 싶다.
4. 기준 품질에 도달한 산출물에 대해 AI 개선 비교표를 고객사에 제출하여 품질 관리 이행을 증명하고 싶다.
5. 반복된 피드백을 통해 자연스럽게 품질 기준을 학습하고, PM 역량을 내재화하고 싶다.
6. 신규/중간 투입 PM도 AI의 가이드를 통해 빠르게 성장할 수 있도록 하고 싶다.

### PMO/품질관리 담당자 관점

7. 전체 프로젝트의 월별, 단계별 품질 점수 추이를 대시보드에서 실시간으로 모니터링하고 싶다.
8. 결함 유형별 분포를 파악하여, 조직 차원에서 자주 발생하는 문제를 선제적으로 대응하고 싶다.
9. PM별 역량 지표(산출물 품질 점수 비교)를 통해, 멘토링이 필요한 PM을 식별하고 싶다.
10. ERROR 발생 빈도, 재검증 횟수 등 리스크 지표를 추적하여 조기 경보를 받고 싶다.
11. 검토 시간 단축 효과를 정량화하여, 비용 절감을 증명하고 싶다.

### 기술 관점

12. Word (.docx), Excel (.xlsx, .csv), HWP (.hwp), PDF (.pdf), 텍스트 (.txt) 등 다양한 파일 형식을 지원받고 싶다.
13. AI가 업로드된 파일을 자동으로 분석하여 산출물 유형(요구사항정의서, 설계서, WBS, 테스트케이스 등)을 인식하고 싶다.
14. 인식된 산출물 유형에 따라 프로젝트 단계(분석/설계/구현/테스트/안정화)가 자동으로 매핑되어, 해당 기준이 적용되기를 원한다.
15. 사내 표준 체크리스트 및 IT SI 품질 기준에 대해 항목별 검토가 수행되기를 원한다.
16. 원본 파일의 구조·포맷·양식이 최대한 유지된 상태에서 개선 원본이 생성되길 원한다.
17. 수정된 부분에 [개선] 태그가 삽입되어, 변경 위치가 명확히 표시되기를 원한다.
18. 개선 원본 파일과 비교표가 Excel 형식으로 자동 생성되어, 즉시 고객사에 제출 가능하길 원한다.
19. 품질 점수가 0~100으로 정량화되어, 단계별 추이를 추적하고 리스크를 조기에 감지할 수 있길 원한다.

### 고객사 검수 관점

20. AI 검토 근거가 문서화된 개선 비교표를 제출받아, 투명하고 신뢰할 수 있는 품질관리 이행을 확인하고 싶다.
21. 각 결함에 대한 구체적인 개선 방향과 사내 기준서 참조가 기록되어, 분쟁 위험을 사전에 예방하고 싶다.
22. 산출물 검수 시 품질 기준을 명확히 알고, 이행 근거를 명확하게 받고 싶다.

## Implementation Decisions

### 파일 처리 및 자동 인식
- **지원 파일 형식**: Word (.docx), Excel (.xlsx, .csv), HWP (.hwp), PDF (.pdf), 텍스트 (.txt)
- **다중 업로드**: 한 번에 여러 파일을 동시에 업로드 지원
- **산출물 유형 자동 인식**: AI가 파일 내용을 분석하여 다음 중 자동 인식
  - 요구사항정의서
  - 설계서 (시스템/DB/인터페이스)
  - WBS (작업 분해도)
  - 테스트케이스 / 테스트 계획서
  - 완료보고서 / 이행완료 보고서
  - 배치정의서
  - 프로그램목록
  - 기타 사내 표준 산출물

### 프로젝트 단계 맵핑 및 품질 기준 적용
- **프로젝트 단계 자동 맵핑**: 인식된 산출물 유형 → 프로젝트 단계(분석/설계/구현/테스트/안정화) 자동 맵핑
- **단계별 품질 기준**: 각 단계에 적합한 품질 기준 및 체크리스트 자동 적용
- **사내 표준 통합**: 사내 표준 체크리스트 및 IT SI 품질 기준을 기반으로 검토

### 품질검토 결과서
- **3단계 등급 부여**:
  - ✋ **ERROR**: 즉시 수정 필수 (요구사항 누락, 심각한 불일치, 필수항목 미기재)
  - ⚠️ **WARNING**: 보완 권고 (품질 기준 미흡, 일관성 오류, 표준 미준수)
  - ✅ **OK**: 기준 충족
- **출력 형식**: Excel 또는 Word 형식
- **포함 내용**: 항목명, 검토 결과, 발견 결함, 근거(사내 기준서 참조), 개선 방향
- **품질 점수**: 0~100점 자동 산출, 정량적 가시화

### AI 개선 원본 파일
- **자동 생성**: AI가 확인된 모든 결함 항목을 직접 수정·보완
- **포맷 유지**: 원본 파일의 구조·포맷·양식을 최대한 유지 (포맷 완벽화는 단계적)
- **변경 위치 표시**: 수정된 부분에 [개선] 태그를 삽입하여 변경 위치 명확히 표시
- **MVP 범위**: Excel (.xlsx) 포맷만 우선 지원 (Word는 Phase 2)
- **PM 활용**: PM이 검토하고 최종 산출물 채택 여부를 판단 (선택적 수용)

### 개선 비교표
- **포함 내용**: 항목명, 심각도 등급(ERROR/WARNING), Before(수정 전), After(수정 후), 개선 사유, 개선자(AI) 및 검토자(PM)
- **출력 형식**: Excel 형식 자동 생성
- **활용 용도**: 고객사 검수 제출 자료, 감리 이행 증빙, 계약 이행 근거, 분쟁 예방 자료

### PM 역량 지원
- **학습 기제**: AI 개선 원본과 비교표를 반복 활용하면서 PM이 품질 기준을 자연스럽게 학습
- **경험 의존도 탈피**: PM 개인 역량 편차를 줄이고 조직 전체의 질적 수준 향상
- **신규 PM 지원**: 경험 부족을 AI 가이드로 보완, 빠른 성장 지원

### 조직 지원 기능
- **PMO 대시보드**: 전체 프로젝트 품질 점수 추이, 결함 유형 분포, PM별 역량 지표, 리스크 지표, 소요 시간 단축 효과 모니터링
- **지속적 고도화**: 반복 검토 이력에서 자주 발생하는 결함 유형 학습, 사내 품질 기준 지속 고도화

## Testing Decisions

### 무엇이 좋은 테스트인가?
- **외부 동작 검증**: 실제 파일을 업로드했을 때 올바른 결과물이 생성되는지 검증
- **구현 세부사항 회피**: 내부 파싱 로직이나 AI 모델 세부사항이 아닌, 사용자가 받는 결과물의 정확도와 품질 검증

### 테스트 대상 모듈

1. **파일 업로드 및 파싱 모듈**
   - 각 파일 형식별 정상 파싱 (Word, Excel, HWP, PDF, TXT)
   - 손상되거나 형식이 잘못된 파일 처리 (에러 메시지 명확성)
   - 다중 파일 동시 업로드 처리

2. **산출물 유형 자동 인식 모듈**
   - 각 산출물 유형이 정확히 인식되는지 검증 (요구사항정의서, 설계서, WBS, 테스트케이스 등)
   - 다양한 형식과 표준의 산출물 인식률

3. **프로젝트 단계 맵핑 모듈**
   - 인식된 산출물 유형이 올바른 단계로 맵핑되는지 검증
   - 단계별 품질 기준이 정확히 적용되는지 검증

4. **품질검토 및 점수 산출 모듈**
   - ERROR/WARNING/OK 분류의 정확도
   - 품질 점수(0~100)의 일관성 및 신뢰성
   - 항목별 결함 설명이 명확하고 구체적인지 검증

5. **AI 개선 원본 파일 생성 모듈**
   - 수정된 내용이 정확하고 원본 포맷이 유지되는지 검증
   - [개선] 태그가 정확한 위치에 삽입되는지 검증
   - Excel 포맷 생성의 정확도 (MVP 범위)

6. **개선 비교표 생성 모듈**
   - Before/After 비교가 정확한지 검증
   - 개선 사유가 명확하고 근거 있는지 검증

### 테스트 전략
- **실제 산출물 기반 테스트**: 사내 표준 산출물들을 테스트 케이스로 사용
- **통합 테스트**: 파일 업로드부터 3종 결과물 생성까지의 End-to-End 프로세스 검증
- **품질 기준 검증**: 사내 IT SI 품질 체크리스트와 비교하여 검토 정확도 측정

## Out of Scope

1. **Word, HWP 편집 기능** (Phase 2)
   - 현재 MVP는 Excel 포맷 개선 원본 생성에 집중
   - Word, HWP는 복잡한 포맷 처리로 인해 향후 단계에서 지원

2. **사용자 권한 관리 및 접근 제어**
   - 팀 내 PM 협업 기능
   - 산출물 공유 및 버전 관리

3. **고급 분석 및 벤치마킹**
   - 조직 간 품질 수준 비교
   - 산업 표준 대비 분석

4. **자동 수정 기능**
   - AI가 최종 산출물을 자동으로 반영하지 않음
   - PM의 검토 후 선택적 수용

5. **실시간 협업 기능**
   - 여러 PM의 동시 편집
   - 댓글/피드백 기능
