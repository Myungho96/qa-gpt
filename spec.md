
# QA-GPT: AI 기반 IT SI 산출물 자동 품질검토 및 개선 플랫폼

 **서비스명** : QA-GPT
 **핵심 슬로건** : 산출물 업로드 → 품질검토 결과서 + 개선 원본 + 개선 비교표 3종 자동 생성
 **팀명** : QA-GPT팀
 **개발 기간** : 1개월 (해커톤)

---

## 1. 문제 정의 및 배경

### 1.1 IT SI 프로젝트의 산출물 품질관리 구조적 한계

**현황:**

* IT SI 프로젝트는 분석→설계→구현→테스트→안정화에 이르는 다단계 산출물이 연쇄적으로 생산
* 각 단계 산출물 간 정합성 오류가 후행 단계에서 누적되는 구조적 취약성
* **산출물 품질 검토는 현재 3가지 방식에만 의존:**
  * ① 제3자 QA 조직
  * ② PMO 검토
  * ③ PM 자체 검토
* **기존 방식의 문제점:**
  * 검토 시점 지연 (제출 후 며칠 뒤)
  * 전수 검토 불가 (비용 제약)
  * 인력·비용 과다
  * **결함의 70% 이상이 요구사항 단계 누락에서 기인**하나, 제출 시점에 실시간 자동 검증 체계가 전무

### 1.2 PM 역량 의존도 심화 및 품질 편차

**현황:**

* 산출물 품질이 담당 PM의 경험·역량에 과도하게 의존
* PM별 품질 편차가 크고 고객사 신뢰도 저하
* 신규/중간 투입 PM의 결함 발생 빈도가 현저히 높음
* 결함 발견 시점이 늦어 수정 비용(Cost of Change) 기하급수적 증가
* 검수·감리 단계에서 뒤늦게 대규모 수정 요구

### 1.3 AI 기술 성숙과 해결 가능성

* LLM(대규모 언어모델) 기술 발전으로 비정형 산출물 문서 파싱·분석·평가·자동 개선 가능
* 단순 검토 결과 알림을 넘어,** ****결함을 직접 수정한 개선 원본과 Before/After 비교표까지 자동 생성 가능**
* 제3자 QA를 AI로 대체하면서 검토 근거가 문서화되어 감리·검수 대응 자료로 직접 활용 가능

## 2. 핵심 기능 (3종 결과물 자동 생성)

### 2.1 산출물 파일 업로드 및 자동 파싱

**지원 형식:**

* Word (.docx)
* Excel (.xlsx, .csv)
* HWP (.hwp)
* PDF (.pdf)
* 텍스트 (.txt)
* **혼합 다중 업로드 지원** (한 번에 여러 파일 동시 업로드)

**자동 인식 기능:**

* AI가 파일 내용을 자동 분석하여 산출물 유형 자동 인식:
  * 요구사항정의서
  * 설계서 (시스템/DB/인터페이스)
  * WBS (작업 분해도)
  * 테스트케이스 / 테스트 계획서
  * 완료보고서 / 이행완료 보고서
  * 배치정의서
  * 프로그램목록
  * 기타 사내 표준 산출물
* **프로젝트 단계 자동 맵핑:**
  * 인식된 산출물 유형 → 프로젝트 단계(분석/설계/구현/테스트/안정화) 자동 맵핑
  * 해당 단계에 적합한 품질 기준 및 체크리스트 자동 적용

---

### 2.2 결과물 ① — 품질검토 결과서 (자동 생성)

**기능:**

* AI가 사내 표준 체크리스트 및 IT SI 품질 기준 대비 항목별 검토 수행
* 각 항목에 대해** ****3단계 등급 부여:**
  * ✋** ****ERROR** : 즉시 수정 필수
    * 요구사항 누락
    * 심각한 불일치
    * 필수항목 미기재
  * ⚠️** ****WARNING** : 보완 권고
    * 품질 기준 미흡
    * 일관성 오류
    * 표준 미준수
  * ✅** ****OK** : 기준 충족
    * 감리·검수 시 근거자료로 활용

**출력 형식:**

* Excel 또는 Word 형식으로 자동 생성
* 항목별 포함 내용:
  * 항목명
  * 검토 결과 (ERROR/WARNING/OK)
  * 발견 결함 내용
  * 근거 (사내 기준서 참조)
  * 개선 방향 (구체적 가이드)

**품질 점수:**

* AI가 자동으로** ****0~100점 산출**
* 프로젝트 품질 수준을 정량적으로 가시화
* 단계별 추이 추적으로 리스크 조기 감지

---

### 2.3 결과물 ② — AI 개선 원본 파일 (자동 생성)

**기능:**

* AI가 품질 결함을 직접 수정·보완한 산출물 새 버전 자동 생성
* **핵심 특징:**
  * 원본 파일의** ****구조·포맷·양식을 최대한 유지** (포맷 완벽화는 단계적)
  * 확인된 모든 결함 항목을 수정
  * 수정된 부분에** ****[개선]** 태그를 삽입하여 변경 위치 명확히 표시
  * **MVP 범위: Excel (.xlsx) 포맷만** (Word는 Phase 2)

**PM 활용 방식:**

* PM은 AI가 생성한 개선 원본을 검토·확인
* 최종 산출물로 채택 여부를 판단 (선택적 수용)
* **PM의 학습과 역량 내재화를 동시에 지원** (왜 이렇게 개선했는지 이유를 통해 학습)

---

### 2.4 결과물 ③ — 개선 비교표 (자동 생성)

**기능:**

* 수정 전(Before)과 수정 후(After)를 항목 단위로 나란히 비교
* Excel 형식으로 자동 생성

**포함 내용:**

* 항목명
* 심각도 등급 (ERROR/WARNING)
* Before (수정 전 내용)
* After (수정 후 내용)
* 개선 사유 (투명한 근거 기록)
* 개선자 (AI) 및 검토자 (PM)

**활용 용도:**

* 고객사 검수 시 제출 자료
* 감리 이행 증빙
* 계약 이행 근거
* 분쟁 예방 자료
* 공식 문서 형태로 즉시 활용 가능

## 3. 서비스 프로세스 — 품질검토 워크플로우

```plaintext
PM 작업 흐름:

[1단계: 산출물 업로드]
  ↓
  파일 선택 (Word/Excel/HWP/PDF/TXT)
  다중 업로드 가능
  ↓

[2단계: AI 자동 검토]
  ↓
  · AI가 산출물 유형 자동 인식
  · 프로젝트 단계 자동 맵핑
  · 해당 기준으로 자동 검토 시작 (수 분 내)
  ↓

[3단계: 3종 결과물 수령]
  ↓
  ① 품질검토 결과서 (ERROR/WARNING 항목 정리)
  ② AI 개선 원본 파일 ([개선] 태그 포함)
  ③ 개선 비교표 (Before/After)
  ↓

[4단계: PM 검토 및 수정]
  ↓
  · 품질검토 결과서로 이슈 파악
  · 개선 원본을 참고하여 최종 산출물 결정
  · 필요시 직접 수정
  ↓

[5단계: 재검증 (무제한)]
  ↓
  · 수정 후 재업로드
  · AI 재검증 실행
  · 품질 점수 개선 확인
  ↓
  · 기준 도달 시: 최종 승인 및 고객사 제출
  · 미달 시: [4단계]로 돌아가 반복
  ↓

[6단계: 개선 비교표 제출]
  ↓
  고객사·PMO에 공식 문서로 제출
  (품질검토 이행 증빙)
```

**지속 지원:**

* PM의 품질 관리 사이클을 독립적으로 수행 가능하도록 지원
* 제3자 QA 의존도 대폭 감소
* 24시간 365일 즉시 검토 가능

## 4. PM 및 조직 지원 기능

### 4.1 PM을 위한 학습 및 역량 내재화

* AI 개선 원본과 비교표를 반복 활용하면서 PM이 품질 기준을 자연스럽게 학습
* 경험 의존도에서 탈피 가능
* 신규 PM의 빠른 성장 지원

### 4.2 PMO/품질관리 담당자용 대시보드

**모니터링 지표:**

* **전체 프로젝트 품질 점수 추이** (월별/단계별)
* **결함 유형 분포** (상위 ERROR 항목 통계)
* **PM별 역량 지표** (산출물 품질 점수 비교)
* **리스크 지표** (ERROR 발생 빈도, 재검증 횟수)
* **소요 시간 단축 효과** (검토 시간 감축 통계)

**활용:**

* 리스크를 조기에 가시화하고 선제적으로 대응
* PM별 역량 격차 파악 및 멘토링 기회 식별
* 조직 전체 품질 수준 개선 추이 추적

### 4.3 조직 지식 자산화

* 반복 검토 이력에서 자주 발생하는 결함 유형 학습
* 사내 품질 기준 지속 고도화
* 조직 전체의 품질 역량이 축적됨
